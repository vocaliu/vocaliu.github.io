Python模拟登陆万能方法-/微博/知乎/-1

全文转载自 1

用到的库有“selenium”和“requests”两个。中心思想就是通过selenium进行模拟登陆，然后将Cookies传入requests，然后用requests进行网站的抓取工作。本方法的优点就是不但规避了“selenium”其本身抓取速度慢的问题（因为仅仅用其作为登陆），又规避了利用requests登陆时需要制作繁琐的Cookies的过程（因为是从selenium直接拿来cookies）。文章前面会列出步骤与代码，后面我还会补充登陆微博与知乎的实例。想要走捷径的童鞋直接看第四部知乎登陆。应该已经不能说是知乎登陆了，因为它适用于所有网站。

需要材料：1.自己喜欢的webdriver （必须） 2.anaconda（可选）
selenium是借助浏览器而运行的，因此需要额外下载一款小型浏览器。我目前用的是chromwebdriver。anaconda推荐大家去下载一个，它里面包含了众多python的库，用起来很方便，而且是免费的！！！

友情链接：google chrome webdriver下载地址 2 楼主在国外测试该链接可以打开，不知国内可否。

第一步：登陆

导入selenium库

from selenium import webdriver
这个地方写上下载的webdriver在电脑中存放的位置，比如我存在了D盘

chromePath = r'D:\Python Program\chromedriver.exe'
将自己存放的位置告诉selenium的webdrive方程去执行（注意：这里面有两个叫做webdriver的东西，一个是selenium里面一个方程的名字，另一个是你刚刚下载的小型浏览器)。记得要对应好webdriver的种类，有多种可选,比如firefox, safari都可以。本次用的是从谷歌下载的webdriver， 所以选择了”.Chrome”（注意：是大写字母）

wd = webdriver.Chrome(executable_path= chromePath)
下面是指挥webdriver为你填写好用户名和密码

wd.find_element_by_xpath('你输入用户名的选项卡的位置信息').send_keys('用户名') wd.find_element_by_xpath('你输入密码的选项卡的位置信息').send_keys('你的密码')
最后指挥webdrive点击登陆按钮，如果是按钮的话选择用click(),如果是表单的话选择submit()

如果不确定是哪一个的话，就每个一个都试试！！！！

wd.find_element_by_xpath('登陆按钮所在位置').click() #如果是按钮 wd.find_element_by_xpath('登陆按钮所在位置').submit() #如果是表单
登陆完成，所有的cookies现在都存在了’wd’里面，可以随时调用。

第二步：将selenium的cookies传入requests

首先导入requests库,并构建Session（这个用来爬网页超级便利）

import reqeusts req = requests.Session()
从刚刚登陆时保存了的‘wd’里面调出它的cookies

cookies = wd.get_cookies()
selenium的cookies并不能直接放在requests里面用，需要做些许调整，但两行代码就足够了。

for cookie in cookies: req.cookies.set(cookie['name'],cookie['value'])
大功告成,现在cookies已经导入requests里面了！可以尝试抓取网页。例如

req.get('需要测试的网站链接')
以上就是python模拟登陆的万能方法，你不需要去分析需要传递给网站的内容。只需要告诉python在什么地方填写用户名与密码然后点击登陆就完成所有操所。十分的便利。下面我会为大家奉上实例，未完待续。。。。。

解释下关键的几个步骤：

1.找位置。推荐使用谷歌浏览器来查找每每个元素的Xpath，参看这个：Chrome浏览器获取XPATH的方法—-通过开发者工具获取 - 飘飘花吹雪的专栏 - CSDN博客。

3.出现验证码需要输入的时候。上面的代码没有写输入验证码的步骤。但是有的时候登陆微博的话，微博会要求输入验证码，这个时候我们可以加一行手动输入验证码的代码。例如：

‘wd.find_element_by_xpath(‘//[@id=”pl_login_form”]/div/div[3]/div[6]/a’).click() #点击登陆
wd.find_element_by_xpath(‘//[@id=”pl_login_form”]/div/div[3]/div[3]/div/input’).send_keys(input(“输入验证码： “))
wd.find_element_by_xpath(‘//*[@id=”pl_login_form”]/div/div[3]/div[6]/a’).click()#再次点击登陆’
输入验证码的时候需要点击两次登陆。因为验证码的输入框只有在点击了一次登陆后才会弹出来！！！根据每个网站的不同而灵活应用selenium是十分重要的！但这个和分析那些Cookies比起来简直是太小儿科了。

第四步：实战，知乎模拟登陆一个傻大笨粗的方法

知乎的登陆实在让人闹心，知乎经常更新，因此即使方法写好了也可能不好用。因此我想到了一个傻大笨粗的终极方法。半手动登陆。仅用selenium打开一个浏览器，然后手动输入账号密码，有验证码就填验证码。等到成功登陆之后使用“get_cookies()”函数来调出它的Cookies。这个方法虽然看起来笨了点，但是效率比那些几百行的代码不知道要高多少！！！

而且你还可以用手机扫描二维码登陆！只要这些登陆了操作是在selenium所打开的浏览器内进行，selenium就可以完全记录下这些Cookies。

req.headers.clear() 是删除原始req里面标记有python机器人的信息。这个信息会被一些网站（比如知乎）捕捉到。造成登陆爬取失败。务必要删除！
